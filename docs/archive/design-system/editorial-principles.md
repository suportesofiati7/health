# Editorial principles

## Hierarchy

Each page has one thesis (`h1`), chapter titles (`h2`) and local subheads (`h3`). Visual size follows meaning rather than element position. Eyebrows orient the reader but do not duplicate full headings.

Recommended content bounds:

| Element | Preferred bound |
|---|---|
| Hero eyebrow | 2–6 words |
| Page heading | 4–14 words; up to 3 lines on 320px |
| Lead | 12–35 words |
| Standard paragraph | 35–90 words |
| Section intro | 1 heading + up to 2 paragraphs before detail |
| Button | 2–5 words where approved content permits |
| Card group | 2–4 peers before using another composition |

Approved source copy is not shortened to satisfy these bounds. When content is longer, layout expands naturally and the issue is recorded rather than hidden.

## Reading rhythm

- Standard body copy is left aligned and capped at `68ch`.
- Policy copy uses the reading container and visible chapter navigation.
- Lists remain lists; ordered processes remain ordered.
- Quotes are interruptions, not a default container for ordinary prose.
- Consecutive sections cannot both use a dense card grid without an editorial reason.

## Bilingual behaviour

Portuguese expansion is expected. Headings use fluid sizes and intrinsic blocks; buttons wrap; navigation switches before labels collide. No manual `<br>` is used to force English line breaks.

Correct: a PT heading wraps to three balanced lines and increases hero height.
Incorrect: reduce PT text below the type scale or clip it to preserve the English screenshot.
