# Franciele Sofiati website

Production source for Franciele Sofiati Biomédica’s bilingual static website. English is served at `/`; Brazilian Portuguese is served at `/pt/`. The site is framework-free: semantic HTML, CSS and native JavaScript modules are compiled into a clean production artifact.

- Canonical origin: `https://www.francielesofiati.com`
- Node.js: 22.19+ (exact development dependencies are locked in `package-lock.json`)
- Production output: `dist/` — generated only; do not edit or deploy the repository root

## Start here

```bash
npm ci
npm run release:check
npm run serve
```

This runs the release validation, builds the deployable artifact and serves it locally. For editable source preview, use `python3 -m http.server 4173`; source partials are fetched at runtime and will not work from `file://` URLs.

Read the [engineering handover](docs/handover.md) for the everyday workflow, [architecture](docs/architecture.md) for ownership, [content workflow](docs/content-workflow.md) for editing, and [deployment guide](docs/deployment.md) before publishing.

## Source of truth

| Concern | Authoritative location |
| --- | --- |
| English page content | Root `*.html` pages |
| Brazilian Portuguese pages | English source plus `data/translation/`, generated by `scripts/generate-portuguese-site.py` |
| Shared interface | `partials/` and `partials/pt-BR/` |
| Styles | `css/site.css` |
| Browser behaviour | `js/main.js` and modules under `js/` |
| Canonical site identity | `data/seo.json` |
| Language route pairs | `data/page-pairs.json` |
| Discovery files | `robots.txt`, `sitemap.xml`, `llms.txt` |
| Deployment output | `dist/`, generated by `npm run build` |

## Key commands

```bash
npm run validate              # assets, PT-BR, SEO and analytics checks
npm run build                 # recreate dist/
npm run release:check         # validate, then build
npm run seo:refresh           # refresh robots/sitemap and write an SEO report
npm run maintain:performance  # weekly performance evidence and recommendations
```

The build composes partials, fingerprints CSS/JS, creates responsive AVIF/WebP variants, minifies pages and writes `dist/_headers`. Deploy only the finished `dist/` directory.

## Editorial and privacy boundary

Keep treatment information educational. Do not publish diagnoses, prescriptions, promises of outcomes, unverified location details, prices or appointment availability. Suitability, timing, settings, recovery and results must remain consultation-dependent.
